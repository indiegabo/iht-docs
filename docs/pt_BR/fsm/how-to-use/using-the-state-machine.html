Usando a Máquina de estados

Contemple: A MÁQUINA DE ESTADOS!!!

Agora que sabemos como configurar um ator e como criar seus estados podemos finalmente usar a Máquina de estados.

O componente StateMachine, diferente dos atores e estados, é um componente pronto para uso. Então você não precisa se preocupar com criar scripts para user ele. Basta anexa-lo a um GameObject, anexar estados ao mesmo GameObject, definir um estado padrão e tudo certo!

But, if you want to understand a little about it… Let’s do it!

Ciclo de vida

Vamos dar uma olhada no ciclo de vida da Máquina de estados:

Nota

A StateMachine na versão atual tem 5 estados: Off, Loading, Ready, On e Paused. Eles são declarados como o enum IndieGabo.FSM.MachineStatus.

Eu gostaria de dar atenção aos seguintes passos:

  1. O primeiro StateMachine.status da máquina é MachineStatus.Off.

Note que, mesmo que a máquina pertença a MonoBehaviour, é o componente Actor que configura a máquina. Porque a máquina só pode começar a operar quando o ator já está acordado (Awake()). Uma vez acordado, o ator chama StateMachine.SetUp(), informando a máquina quem ela deve tratar como o seu ator.

  1. Nesse ponto, a máquina irá mudar StateMachine.status de MachineStatus.Off para MachineStatus.Loading.

Aqui, a máquina irá carregar todos os estados anexados ao seu GameObject e, para cada estado, chamar seu State.OnLoad() seguido do State.SortTransitions(). Esse último método garante que as transições sejam ordenadas de acordo com suas prioridades. Vem daí a importância de registrar qualquer StateTranstion do seu estado dentro de seu State.OnLoad().

  1. Agora, a máquina irá mudar StateMachine.status de MachineStatus.Loading para MachineStatus.Ready.

Uma vez pronta (ready), a máquina irá definir o StateMachine.defaultState como o o StateMachine.currentState. Lembra que você deve definir o StateMachine.defaultState usando o inspetor da Unity.

  1. A máquina irá finalmente transitar o StateMachine.status de MachineStatus.Ready para MachineStatus.On e está rodando!!! Boa!!

Métodos Tickers e os status da Máquina

Uma vez que a máquina está ligada, significando que o StateMachine.status é MachineStatus.On, ela irá chamar os Tickers do StateMachine.currentState de acordo com os “Frame Handlers” do ator. Enquanto o StateMachine.status for MachineStatus.On, ela irá proceder desta forma.

Se em qualquer momento você precisar pausar este comportamento, você pode chamar StateMachine.Pause() e o StateMachine.status será definido como MachineStatus.Paused. Desta forma, nenhum dos Tickers será passado para o StateMachine.currentState, portanto, podemos considerar que a máquina está pausada.

// Some other code from inside a state

 public void Tick()
 {
   machine.Pause(); // Pausing the machine
 }

 // Some other code

Para continuar, simplesmente chame StateMachine.Resume() e o StateMachine.status será definido como MachineStatus.On.

// Some other code from inside a state

 public void Tick()
 {
   machine.Resume(); // Resuming the machine.
 }

// Some other code

Configurando a máquina

Você pode simplesmente criar um GameObject e anexar o StateMachine componente dentro dele. Isto também vale para o Actor. Veja:


Mas, como o GameObject do ator tende a ter múltiplos componentes anexados a ele, para fins de organização, acho mais conveniente proceder criando um GameObject para o ator e usar a máquina em um GameObject filho do GameObject do ator.


Ambas as estratégias funcionarão transparentemente para você. Fique a vontade para escoher.

Como observamos, eu preparei um item de menu para você para colocar uma nova máquina de estado em sua hierarquia. Mas se você desejar anexar o componente StateMachine em qualquer GameObject, você pode procurar por ele usando o botão “Add Component” do inspetor.
