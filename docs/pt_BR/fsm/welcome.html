FSM

Muito boas vindas a FSM do IndieGabo!

Esta é uma solução de FSM (Finite State Machine, ou, Máquina de estados Finita) baseada em código para seu projeto Unity.

O que é uma Máquina de Estados Finita?

A Máquina de Estados Finita é um padrão de projeto focado em lidar com entidades que possuem vários estados possíveis.

Porquê esta FSM?

Embora saibamos que o Animator da Unity é uma Máquina de Estados, ele pode ficar confuso para se usar como um gerenciador de ações de um modo geral. Esta FSM lhe ajuda a organizar seu código mantendo a lógica das ações dos atores dentro do seu respectivo estado (vamos falar sobre atores em alguns instantes).

Benefícios práticos da utilização desta FSM:

  • Te ajuda a manter tudo organizado

  • Uma vez que você entenda como funciona, você vai economizar horas de depuração, pois você vai saber exatamente onde cada ação está sendo executada

  • Evita sobrecarga de if statements, pois cada estado só verificará as condições baseadas nas transições registradas

  • 5 minutos pra entender como funciona e colocar pra rodar

Como funciona

Conside um ator qualquer coisa no seu jogo que realiza várias ações. A máquina de estados então é configurada para regular os estados de ações de um ator. Os estados do ator devem ser registrados na máquina de estados, sendo que um estado sabe exatamente quando e para qual outro estado transitar sob condições específicas.

Além disso, para o uso desta FSM, consideramos que um estado não só é um tipo de status ou definição. Os estados realmente executam o que o ator deve fazer enquanto em um determinado estado de ações.
